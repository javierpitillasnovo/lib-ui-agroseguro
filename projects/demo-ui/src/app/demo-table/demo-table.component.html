Código a incluir en TS para el correcto funcionamiento

Imports
import { NgTemplateOutlet } from '@angular/common';
import { tableDirective, UiTableComponent } from '@agroseguro/lib-ui-agroseguro';

Dentro del decorador @Component incluir:
imports: [tableDirective, UiTableComponent, NgTemplateOutlet]

Código de ejemplo HTML

<ui-table title="Titulo de la tabla" [dataOriginal]="ArrayDeData" [filterId]="'id'" [showAddButton]="true" [defaultActions]="'checkbox'" [showPagination]="true" (itemSelected)="itemSelected($event)" (itemAdd)="itemAdd($event)">
<ng-template tableColumn header="NombreDeUnHEader" [headerIsVisible]="true" let-item>
{ { item.nombrevariable } }
</ng-template>
</ui-table>

Propiedades Input()

title : Input Título de la tabla.
[dataOriginal] *: Input data de la tabla.
[filterId] : Propiedad de id de la data que luego el componente usará para filtrados y el correcto funcionamiento. Evitando que back deba de devolver la propiedad siempre como id
[defaultActions] : Boolean para mostrar o no los chebkbox o radios, acepta los siguientes valores: false | 'checkbox' | 'radio'
[showAddButton] : Boolean para mostrar o no el botón de añadir.
[showPagination] : Boolean para mostrar o no la paginación.
totalItems : Number con los items totales.
hasError : Boolean para activar las clases de error.

Propiedades Model

[(pageSize)] : Model de la cantidad de items por página
[(currentPage)] : Model del total de items

Propiedades Output()

(itemSelected) : Evento al que suscribirnos, nos pasa el id y estado del checkbox al
seleccionar una fila
(itemAdd): Disponible cuando [defaultActions] es 'checkbox' o 'radio'.
Evento al que suscribirnos, nos dice que item se ha seleccionado, válido para checkbox o
radiobuttons
(paginationAction): Evento que emite al hacer click en el icono de página siguiente o anterior.
